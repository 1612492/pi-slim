# Explorer Recon

Use this skill when creating or interpreting an `explorer` child Pi run.

## Explorer scope

- local code search only
- no editing
- no external docs lookup
- compact findings with file paths and line references when possible

## Good explorer tasks

- find entry points
- trace symbol usage
- locate config and integration boundaries
- identify likely files for implementation

## Bad explorer tasks

- broad implementation work
- code rewriting
- external API research

## Output shape

- short summary
- relevant files
- important symbols or flows
- risks or unknowns
